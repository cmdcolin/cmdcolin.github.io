<p>It is common in typescript to write <code>Record&#x3C;string,X></code> for whatever type <code>X</code> you
store in an object.</p>
<p>But, are you sure you're actually going to get an <code>X</code> back? What if some random
user provided string is provided and it produces undefined? You might think, as
a typescripter "do I really have to add undefined to the signature, shouldn't
Typescript check this for me"? Well..</p>
<h2 id="nouncheckedindexedaccess"><a aria-hidden="true" tabindex="-1" href="#nouncheckedindexedaccess"><a href="#nouncheckedindexedaccess" style="margin-right: 10px">#</a></a>noUncheckedIndexedAccess</h2>
<p>There is actually a built-in Typescript setting called
<a href="https://www.typescriptlang.org/tsconfig#noUncheckedIndexedAccess"><code>noUncheckedIndexedAccess</code></a>
that will basically change your usage of a <code>Record&#x3C;string,X></code> into a
<code>Record&#x3C;string,X|undefined></code>. It actually makes the return type of accessing
both objects and arrays into an <code>X|undefined</code>.</p>
<p>The catch?</p>
<ul>
<li>Very few people use this setting</li>
<li>It is default false</li>
<li>It is probably too "annoying" (or produces too many "false positives") for
most projects to warrant it being turned on</li>
</ul>
<p>Therefore: you really do have to ask yourself. Do you feel lucky punk? Sorry
wrong line. I mean, do you want to use a <code>Record&#x3C;string,X></code> or a
<code>Record&#x3C;string,X|undefined></code>?</p>
<p>Enjoy! <a href="https://www.typescriptlang.org/tsconfig#noUncheckedIndexedAccess">https://www.typescriptlang.org/tsconfig#noUncheckedIndexedAccess</a></p>
<p>Discussion on why this is not on by default:
<a href="https://github.com/microsoft/TypeScript/issues/49169">https://github.com/microsoft/TypeScript/issues/49169</a></p>
<h2 id="footnote-should-nouncheckedindexedaccess-be-split-into-two-settings"><a aria-hidden="true" tabindex="-1" href="#footnote-should-nouncheckedindexedaccess-be-split-into-two-settings"><a href="#footnote-should-nouncheckedindexedaccess-be-split-into-two-settings" style="margin-right: 10px">#</a></a>Footnote: Should <code>noUncheckedIndexedAccess</code> be split into two settings?</h2>

Records, one for Arrays. A common complaint is that the 'array access' part of
