I see you don't have noUncheckedIndexedAccess:true. Maybe you should use Record<string, X|undefined>

2023-09-27

It is common in typescript to write Record<string,X> for whatever type X you store in an object (or {[key:string]:X}).

But, are you sure you're actually going to get a string back?

There is an actually quite cool Typescript setting called noUncheckedIndexedAccess that will always make the return type of accessing of both OBJECTS AND ARRAYS X|undefined

The catch?

  • Very few people use this setting
  • It is default false.
  • It is probably too "annoying" for most projects to warrant it being turned on

Therefore: ask yourself. Do you have a Record<string,X> or Record<string,X|undefined>?

In my current project, turning it on produces [10:06:14 PM] Found 1006 errors. I doubt we would turn in on for the project, so I think that I will need to actively brainstorm, when I make that record type, should it include undefined in the signature.

Enjoy! https://www.typescriptlang.org/tsconfig#noUncheckedIndexedAccess

Discussion on why this is not on by default: https://github.com/microsoft/TypeScript/issues/49169
