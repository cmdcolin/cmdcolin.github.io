I see that noUncheckedIndexedAccess is false. Maybe you should use Record<string, X|undefined>

2023-09-27

It is common in typescript to write Record<string,X> for whatever type X you store in an object.

But, are you sure you're actually going to get an X back? What if some random user provided string is provided and it produces undefined? You might think, as a typescripter "do I really have to add undefined to the signature, shouldn't Typescript check this for me"? Well..

#noUncheckedIndexedAccess

There is actually an awesome Typescript setting called noUncheckedIndexedAccess that will basically change your usage of a Record<string,X> into a Record<string,X|undefined>. It actually makes the return type of accessing both objects and arrays into an X|undefined.

The catch?

  • Very few people use this setting
  • It is default false
  • It is probably too "annoying" (or produces too many "false positives") for most projects to warrant it being turned on

Therefore: you really do have to ask yourself. Do you feel lucky punk? Sorry wrong line. I mean, do you have a Record<string,X>... or do you need to include the undefined Record<string,X|undefined>?

In my current project, turning it on produces [10:06:14 PM] Found 1006 errors. so I doubt we would turn in on for the project, so I think that I will need to actively brainstorm, when I make that record type, should it include undefined in the signature.

Enjoy! https://www.typescriptlang.org/tsconfig#noUncheckedIndexedAccess

Discussion on why this is not on by default: https://github.com/microsoft/TypeScript/issues/49169
