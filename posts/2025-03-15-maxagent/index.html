Maximizing agentic coding

2026-03-15

The “Ralph loop” is a popular workflow but nobody explains it plainly. The repo at https://github.com/snarktank/ralph says:

“Ralph is an autonomous AI agent loop that runs repeatedly until all PRD items are complete.”

It references the term PRD over and over without ever defining it. So..what is it?

A PRD (“project requirements document”) is…or can be… just a todo list

A markdown file with checkboxes works fine:

- [ ] Add priority column to tasks table
- [ ] Show priority badge on task cards
- [ ] Add priority selector to task edit modal
- [ ] Filter tasks by priority, persist in URL

The agent reads this at the start of each session, picks the top unchecked item, implements it, and checks it off. If you have a scrappy handwritten todo list, just paste it and ask Claude to clean it up. The format doesn’t matter.

You tell it to solve things on the list, and check items off one by one. You can tell Claude to reorganize your hand-written todo list into something that looks less like a chaotic stream of consciousness from your brain, but ultimately that’s all there is to the PRD doc.

But this is important: To really lean into this, you let the AI also manage your TODO list. So as it is working, it will be reading the TODOs, and writing what it is done, and adding further TODOs even.

How does this connect to the “Ralph loop”

The Ralph loop is a shell script where each iteration spawns a fresh Claude instance and pipes in a CLAUDE.md as the prompt. That CLAUDE.md contains instructions like “read the PRD at PRD.md” and “pick the highest priority incomplete task”, so it’s instructions pointing to more instructions. The agent does one task, commits, and exits. Then the loop spawns a new instance for the next task.

for i in $(seq 1 $MAX_ITERATIONS); do
  OUTPUT=$(claude --dangerously-skip-permissions --print < CLAUDE.md)
  if echo "$OUTPUT" | grep -q "<promise>COMPLETE</promise>"; then
    exit 0
  fi
done

The agent has no idea there’s a loop. It stops after one task because the CLAUDE.md prompt (at least the one ralph has https://github.com/snarktank/ralph/blob/main/CLAUDE.md#important) tells it to:

Work on ONE story per iteration. If stories remain, end normally.
If all are complete, reply with: <promise>COMPLETE</promise>

After each run, the bash script greps the output for <promise>COMPLETE</promise>1. If found, the loop exits. Otherwise it spawns another instance. The entire stop condition is a grep.

Each instance starts with zero context. Memory between iterations comes from git history and the updated todo list. Claude Code also has auto-memory which accumulates codebase knowledge across sessions, so the hand-rolled progress.txt that ralph ships with isn’t really necessary anymore.

You be the Ralph

You don’t have to use the Ralph repo or even the Ralph bash loop to get most of the benefits

Just use Claude like usual but start each session with: “read todo.md, pick the top incomplete task, implement it, mark it done, then stop.” Review the result, new session, repeat.

Then you can also type in the Claude interface to steer it mid-task and do whatever else you do with Claude like normal

Amazing

Conclusion

The ralph repo adds shell scripts, “skills”, a JSON format for its PRD, a progress log, and more. Things that seem necessary one month aren’t needed the next.

I’m not using this ralph loop stuff yet, but just figuring out this workflow for having the AI manage a TODO list I think gives you about 90% of the benefits, and gets you closer to faster agentic iteration.

If I’m missing something let me know :)

Footnotes

  1. The <promise> tag is unexplained in the repo. It’s just an arbitrary signal string for the grep. Could be anything as long as the agent wouldn’t accidentally produce it in normal output. ↩
