Maximizing agentic coding, or, wtf is ralph

2026-03-15

The “Ralph loop” is a popular workflow but nobody, at least to me, seems to explain it in laymans terms, which seems ironic for the finger-in-nose logo.

And the repo says stuff like this:

“Ralph is an autonomous AI agent loop that runs repeatedly until all PRD items are complete.”

It references the term PRD over and over without ever defining it, or even trying to define it.

I’ve worked in software for many years now (ow my back) and don’t come across the word PRD…So wtf is it?

A PRD (“project requirements document”) is just a todo list

Your “PRD” can just be a markdown file with checkboxes like a normal TODO list:

- [ ] Add priority column to tasks table
- [ ] Show priority badge on task cards
- [ ] Add priority selector to task edit modal
- [ ] Filter tasks by priority, persist in URL

But this is the important part: you let the AI also manage your TODO list. So as it is working, it will be reading the TODOs, and writing what it has completed or discovered, and adding further TODOs.

Ok so how does this connect to the “Ralph loop”

The Ralph loop is a shell script like this

while true; do
  OUTPUT=$(claude --dangerously-skip-permissions --print < CLAUDE.md)
  if echo "$OUTPUT" | grep -q "<promise>COMPLETE</promise>"; then
    exit 0
  fi
done

so it’s a bash loop that just keeps running Claude until the literal string <promise>COMPLETE</promise>1. If found, the loop exits. Otherwise it keeps going another instance.

Each instance starts with zero context. Memory between iterations comes from git history and the updated todo list. Claude Code also has auto-memory which accumulates codebase knowledge across sessions, so the hand-rolled progress.txt that ralph ships with isn’t really necessary anymore.

You be the Ralph

You don’t have to use the Ralph repo or even the Ralph bash loop to get most of the benefits

Just use Claude like usual but start each session with: “read todo.md, pick the top incomplete task, implement it, mark it done, then stop.” Review the result, new session, repeat.

Then you can also type in the Claude interface to steer it mid-task and do whatever else you do with Claude like you do normally.

Conclusion

This might be a obvious post for some. I dunno. I think figuring out this workflow is just an AI managing a TODO list is a good mindset.

You do not have to use the ralph github repo to get the benefits of this, just tell Claude manually to read your todo list file and work on it.

If you want to go into danger mode you can run the ralph loop bash script, but I don’t even do that. I even break the rules and let it solve more than one PRD item per session. Gasp!

If I’m missing something let me know. Enjoy :)

Footnotes

  1. The <promise> tag is unexplained in the repo. It’s just an arbitrary signal string for the grep. Could be anything as long as the agent wouldn’t accidentally produce it in normal output. ↩
