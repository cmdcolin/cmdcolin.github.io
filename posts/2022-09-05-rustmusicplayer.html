Creating a music player using Rust/GTK4 - fml9000

2022-09-05

I have started endeavoring to learn Rust. I did the rustlings exercises but I knew the concepts would fade rapidly from my brain without some practice. I have now started making a music player using Rust+GTK4.

I chose GTK4 is to have a linux native GUI to scratch the itch that I've had in lacking foobar2000 on linux. I have a particular foobar2000 setup that uses default UI with four quadrants. I have used foobar2000 under wine and it's not terrible, but it has background CPU consumption of about 14% idle and is just a little "off". I have also used a variety of other GUI (quodlibet) and CLI (cmus) apps on linux, but they didn't really feel quite right.

#Hurdles experienced

#Choosing a tech stack

I started by attempting with Relm4, which I may return to at some point, but trying to juggle learning Relm4-style widgets, GTK, and rust all at once was a bit much. I stepped to using gtk4-rs directly.

#Trying to create a "spreadsheet style" or data grid of tracks

Then, I wanted to create a data grid showing to display e.g. a table with artist, track, album, etc. I chose to use the Gtk::ColumnView to drive this. I stumbled around looking for example code, but there was none specifically for the ColumnView. Soon, I realized the ListView example from the gtk4-rs examples with it's factory function was similar in concept.

#How to store data in a ListStore

Then, I wanted to get the data for the songs into a ListStore, but I thought if I could make my own GObject subclass, it would solve everything, but I had trouble getting this. Finally, I randomly stumbled on this link using a BoxedAnyObject with a good example of storing data in a ListStore https://gtk-rs.org/gtk-rs-core/git/docs/glib/struct.BoxedAnyObject.html

The BoxedAnyObject is a GObject, so my troubles got automatically solved essentially

#Accessing the ListStore in the ColumnView factory functions

There was no example code for this so I stuggled with this (for a couple days) before realizing that the ListView example in the gtk4-rs codebase has similar concepts (factory to create grid cells, etc). I created a PR (https://github.com/gtk-rs/gtk4-rs/pull/1111) to demonstrate simple ColumnView usage.

After this hurdle was crossed, I felt much more comfortable in the GTK mindset, and I quickly fleshed out some more UI for the app
